What is the Difference between Var, let and Const ?
Var is the old way of declaring variables in JavaScript — it is function-scoped and can be re-declared or updated, which often causes bugs. let is the modern way and is block-scoped, meaning it only works inside the { } where it is declared; it can be updated but not re-declared in the same scope. const is also block-scoped but cannot be updated or re-declared; it is used for values that should never change. In simple words: use let for variables that will change, use const for fixed values, and avoid var because it behaves unpredictably.
What is the Difference Between map() , ForEach() and Filter() ?
ForEach() is used when you want to loop through every item in an array and perform an action, but it does not return anything. map() also loops through an array, but it creates and returns a new array based on whatever transformation you apply to each item. On the other hand, filter() goes through the array and returns a new array containing only the elements that meet a specific condition. In summary: use forEach when you just want to do something, use map when you want a new transformed array, and use filter when you want to keep only certain items based on a condition.
What are arrow Functions and what is the Difference between arrow functions and Normal Functions ?
Arrow functions are a shorter and more modern way to write functions in JavaScript. They use the => symbol and make the code look cleaner, especially for small tasks. Unlike normal functions, arrow functions do not have their own this, arguments, super, or new.target, which means they behave differently when used inside objects, classes, or event handlers. Arrow functions cannot be used as constructors (you cannot create objects with new using them), but normal functions can. Normal functions also get hoisted, meaning they can be used before they are written, while arrow functions cannot. In short: arrow functions are shorter and good for simple callbacks, while normal functions are more powerful and flexible for object methods, constructors, and places where this matters.
How do the Java Script Promisses are work ?
A Promise in JavaScript is like a guarantee that some work (usually something slow like fetching data from an API) will finish in the future. Instead of stopping the whole program, JavaScript continues running and the Promise tells you when the work is done. How does Clauser Works in Java Script ?
A Promise in JavaScript represents a value that will be available now, later, or never. It has three states: pending, fulfilled, and rejected. Promises help you handle asynchronous tasks like API calls. When the task succeeds, the promise becomes fulfilled and runs the .then() block; if it fails, it becomes rejected and runs the .catch() block. This makes asynchronous code cleaner and avoids “callback hell.” Promises simply say: “I’m doing something; when it’s done, I’ll let you know.”
